# Qubika Club Administration — E2E Tests

This repository contains an end-to-end Playwright test that registers a test user, logs into the web app, and exercises category/subcategory flows.

Files of interest:
- [e2e/exercise2.e2e.spec.ts](e2e/exercise2.e2e.spec.ts) — contains the test [`Execute complete test E2E`](e2e/exercise2.e2e.spec.ts).
- [playwright.config.ts](playwright.config.ts) — Playwright configuration for the project.
- [package.json](package.json) — project scripts and dependencies.
- [credentials.json](credentials.json) — generated by the test at runtime with created user credentials.
- [.vscode/launch.json](.vscode/launch.json) — VS Code debug configuration.
- [.gitignore](.gitignore) — ignored files.

Overview of the test
- The test implemented in [e2e/exercise2.e2e.spec.ts](e2e/exercise2.e2e.spec.ts) registers a new user via API, saves payload to [credentials.json](credentials.json), opens the UI, logs in with the created user, then creates a category and a subcategory. See the test entry [`Execute complete test E2E`](e2e/exercise2.e2e.spec.ts) for the full flow.
